Broken Link Checker Premium

⭐

This feature requires a Premium Core licence. Activate →

Automatically scan all your posts and pages for broken external links and fix them before visitors notice.

Running a Scan

Trigger a scan from the command line:

php spark links:check

The command checks every external link in published posts and pages, records the HTTP status, and stores results in the database.

Viewing Results

Go to Admin → Broken Links. The table shows:

  • URL — the broken link
  • Status — HTTP response code (e.g. 404, 500, timeout)
  • Found in — which post or page contains the link
  • Last checked — date of the most recent check

Re-Checking Individual Links

Click Recheck next to any link to run a fresh HTTP check immediately. If the URL has been fixed, it will be removed from the broken list.

Dismissing False Positives

Some links may report errors due to anti-scraping measures (blocking headless requests). Click Dismiss to mark a link as intentionally ignored — it won’t appear in future reports.

Weekly Cron

Add a weekly cron job to automate scanning (Sundays at 2 AM):

0 2 * * 0 /usr/bin/php /path/to/pubvana/spark links:check >> /dev/null 2>&1
