Backup & Export Premium

⭐

This feature requires a Premium Core licence. Activate →

Create and download a complete backup of your site — database and uploaded files — in a single ZIP archive.

Creating a Backup

  1. Go to Admin → Backup
  2. Click Create Backup Now
  3. Pubvana dumps the database and zips the public/uploads/ directory
  4. The backup appears in the list when complete (usually a few seconds)

Downloading a Backup

Click the Download button next to any backup in the list. The ZIP file contains:

  • database.sql — full MySQL dump
  • uploads/ — all media files
  • backup-info.txt — timestamp, site URL, and Pubvana version

Deleting Old Backups

Click the Delete button next to any backup to remove it from the server. Backups are stored in writable/backups/ — you can also manage them via FTP/SFTP.

Recommended Schedule

  • Active blogs: daily automated backup (via external cron + curl ping, or a hosting cPanel task)
  • Low-traffic sites: weekly manual backup before any significant changes
Off-site storage Download backups to your local machine or cloud storage. Server-side backups are not a substitute for off-site copies.
