Redirects

Use redirects to send visitors from old URLs to new ones — preserving SEO value and preventing broken links.

Creating a Redirect

  1. Go to Tools → Redirects
  2. Click New Redirect
  3. Enter the Source (the old URL path, e.g. /old-post-slug)
  4. Enter the Destination (the new URL path or full URL)
  5. Choose the type: 301 Permanent or 302 Temporary
  6. Click Save

301 vs 302

TypeWhen to use
301 PermanentThe URL has moved forever (changed slug, retired page). Passes SEO ranking.
302 TemporaryShort-term redirect (A/B test, seasonal page). Does not pass SEO ranking.

Common Use Cases

  • You changed a post’s slug after it was indexed by Google
  • You deleted a page and want to point visitors to a replacement
  • You’re migrating from WordPress and old URLs differ from new ones

Wildcard vs Exact Match

By default, redirects match the exact source path. Tick Wildcard to match any URL that starts with the source path — useful for redirecting an entire section (e.g. /blog/* → /posts/*).
